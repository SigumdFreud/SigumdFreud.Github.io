    存储引擎

    –mysql特有的。是指一个表组织数据的方式。

    1,MyISMA引擎 -(格式文件，数据文件，索引文件)可压缩，

    create table tt(

    id int primary key,

    sname varchar(255)

    )engine=MyISMA default charset=utf-8;

    2，InnoDB(默认存储索引，非常安全，支持事务，效率高，不能很好的节省存储)

    3，MEMORY存储索引(所有表都是存在.frm格式的文件中，查询效率高，关机后，数据消失)

    事务（重中之重）

    一个完整的业务逻辑。

    转账，A给B 1000元，A 减去1000元，B加上1000元，两者必须同时发生，或者同时不发生。

    一个事务就是多条DML语句的执行，这些语句要么同时成功。要么同时失败。

    提交事务commit：数据彻底更改，事务是成功运行

    回滚事务rollback：恢复原来的数据，即所有的DML语句全部撤销。

    特性

    原子性：最小的工作单元

    一致性：必须同时成功

    隔离性：有一定的距离

    持久性：提交后的最终保障，将数据直接保存到硬盘中

    隔离级别

    读未提交：可以读到脏数据。（最低的隔离级别）

    读已提交：只能读取提交后的数据。不可重复读取。

    可重复读：读到的是幻影，不真实。

    序列化：最高级别
