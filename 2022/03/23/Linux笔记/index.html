    权限

    0位

    ​ I 链接 d 文件夹 c 设备 b 块设备（硬盘） -普通文件

    1-3位

    ​ 所有者拥有该文件的权限

    4-6位

    ​ 所属组拥有该文件的权限

    7-9位

    ​ 其他用户拥有该文件的权限

    修改权限

    u:所有者g:所有组o:其他人a:所有人(u、g、o的总和)

    1. chmod u=rwx,g=rx,o=x 文件/自录名（修改三个的权限）

    2. chmod o+w 文件/目录名（给其他人加上w权限）

    3. chmod a-x 文件/目录名（给所有人减去x权限）

    ·
    第二种方式:通过数字变更权限:
    r=4 w=2 x=1 rwx=4+2+1=7
    chmod u=rwx,g=rx,o=x 文件目录名

    at定时任务调度

    at命令是一次性定时计划任务，at的守护进程atd会以后台模式运行，检查作业队列来运行。
    默认情况下，atd守护进程每60秒检查作业队列，有作业时，会检查作业运行时间，如果时间与当前时间匹配，则运行此作业。
    at命令是一次性定时计划任务，执行完一个任务后不再执行此任务了
    在使用at命令的时候，一定要保订atd过程的启动，可以使用相关指令来查看。

    ps -ef | grep atd 查看是否有守护进程

    ps详解
    1.指令:ps -auxlgrep xxx ,比如我看看有没有sshd服务

    2.指令说明
    . System V展示风格. USER:用户名称.PID:进程号
    ·%CPU:进程占用CPU的百分比
    ·%MEM:进程占用物理内存的百分比
    · VSZ:进程占用的虚拟内存大小(单位:KB)

    · RSS:进程占用的物理内存大小(单位:KB )·TT:终端名称缩写.
    STAT:进程状态，其中S-睡眠，s-表示该进程是会话的先导进程，N-表示进程拥有比普通优先级更低·的优先级，R-正在运行，D-短期等待，Z-僵死进程，T-被跟踪或者被停止等等
    .STARTED:进程的启动时间
    · TIME :CPU时间，即进程使用CPU的总时间
    .COMMAND:启动进程所用的命令和参数，如果过长会被截断显示

    rpm包

    rpm用于互联网下载包的打包及安装工具，它包含在某些Linux分发版中。它生成具有.RPM扩展名的文件。RPM是RedHat Package Manager(RedHat软件包管理工具）的缩写，类似windows的
    setup.exe，这一文件格式名称虽然打上了RedHat的标志，但理念是通用的。

    rpm -qa 查询所安装的所有rpm软件包

    rpm -q firefox 查询火狐是否安装

    rpm -qi firefox 查询具体的信息

    rpm -ql firefox 查询具体的位置

    rpm -qf 文件全路径名 查询该路径下有没有软件包

    卸载rpm包

    rpm -e firefox 卸载firefox软件包

    1.如果其它软件包依赖于您要卸载的软件包，卸载时则会产生错误信息。如:$ rpm -e foo removing these packages would break dependencies:foo is needed by bar-1.0-1
    如果我们就是要删除foo这个rpm包，可以增加参数–nodeps ,就可以强制删除，但是一般不推荐这样做，茵为依赖于该软件包的程序可能无法运行
    如: $ rpm -e –nodeps foo

    安装rpm包

    rpm -ivh rpm包全路径名称 //i（install）v（version）h（hash）

    拓展：1.rpm -U可以更新rpm软件包 2.rpm –rebuilddb可以重建rpm数据库 3.rpm –allfiles可以安装或升级包里所有的缺失文件,不管其是否存在.

    yum

    Yum是一个Shell前端软件包管理器。基于RPM包管理，能够从指定的服务导自动下载RPM包并且安装，可以自动处理依赖性关系，并且一次安装所有依赖的软件包。
